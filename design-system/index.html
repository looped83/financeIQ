FinanceIQ Design System

The tokens, type scale and components that make up the FinanceIQ analytics dashboard — a dense, dark-first fintech surface for reading personal finance data at a glance. Every value below is extracted from the shipping app; click any swatch to copy it.

Dark-first Cool / deep-navy Data-dense 12-color series ramp
01

Color

A cool, near-black navy foundation with a blue→purple brand pairing. Semantic colors double as data-encoding: green is money in, red is money out — a mapping that must stay stable across charts, tables and KPIs.

DoKeep green→in / red→out fixed. Reach for the ramp only for nominal categories (merchants, asset classes) where hue carries no value meaning.
Don'tInvent one-off hex values. If a color isn't a token, it doesn't belong — the palette is capped on purpose so nothing competes with the semantic set.

16

Chart theming

Chart.js defaults, centralized in src/charts/chartTheme.ts. Axes use faint white grid lines and muted tick labels so data ink dominates; series colors come from the semantic set (single-metric) or the categorical ramp (breakdowns).

Grid linesrgba(255,255,255,.05)
Tick labels#8b9bb5 · 10px (--text-muted)
Legend labels#b0bfd0 · 11px (--text-dim)
Y-axis formatfmtN(v) + '€'
Single-metric--income / --expense / --invest / --dividend
BreakdownsPAL[i % 12] (categorical ramp)
Bar radius4px · borderRadius
Fillsrgba(color, .75) for bars, solid for donuts
DoRoute every new chart through BASE + darkAxes() so grid, ticks and legend stay uniform across all ten tabs.
Don'tSet per-chart tick or grid colors by hand — that's how the axes drift out of sync tab-to-tab.
